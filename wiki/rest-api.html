REST API Reference

The complete catalogue of HTTP endpoints exposed by a slskd daemon. Every controller action is documented below with its HTTP method, full path, authorization policy/role, path and query parameters, request and response shapes, and notable status codes. All routes are versioned under /api/v0, accept and return application/json with camelCase property names, and are authenticated as described in Authentication & Security. For the live, machine-readable contract, slskd also serves OpenAPI/Swagger documents under /swagger.

Conventions

Base path & versioning

Every controller is mapped with the route template api/v{version:apiVersion}/[controller] and declares [ApiVersion("0")]. The current and only API version is therefore v0; the effective base path is /api/v0. Throughout this page, endpoint paths are written relative to that base (for example, /searches means /api/v0/searches). The two telemetry controllers use the nested template api/v{version:apiVersion}/telemetry/[controller], yielding /api/v0/telemetry/metrics and /api/v0/telemetry/reports.

Source: src/slskd/**/API/**Controller*.cs ([Route], [ApiVersion] attributes)

Content type

Controllers declare [Produces("application/json")] and (except the telemetry controllers and RelayController) [Consumes("application/json")]. Request bodies must be JSON; even a "plain string" body — for example the body of POST /conversations/{username} — is a JSON-encoded string ("hello", with the quotes), not raw text. Responses are JSON unless otherwise noted (the metrics endpoint can emit Prometheus text/plain; binary downloads use application/octet-stream).

JSON casing

DTOs are serialized with camelCase property names. A C# property such as BytesTransferred appears on the wire as bytesTransferred; Username as username. The web client and the Swagger generator both assume camelCase (Swagger is configured with DescribeAllParametersInCamelCase()). Enum values, however, are generally serialized as-declared (PascalCase strings such as "Download", "Succeeded") and are matched case-insensitively on input.

Source: src/slskd/Program.cs, src/web/src/lib/*.js

Authorization

Each action is decorated with [Authorize(Policy = …, Roles = …)] or [AllowAnonymous]. Three policies and four roles are defined:

ConstantValueMeaning
AuthPolicy.Any"Any"Satisfied by a valid JWT or a valid API key.
AuthPolicy.JwtOnly"Jwt"Requires a JWT bearer token (not an API key).
AuthPolicy.ApiKeyOnly"ApiKey"Requires an API key (used by relay controller endpoints).
AuthRole.Any"ReadOnly,ReadWrite,Guest,Administrator"Any authenticated role.
AuthRole.AdministratorOnly"Administrator"Administrator role only.
AuthRole.ReadWriteOrAdministrator"ReadWrite,Administrator"ReadWrite or Administrator.
AuthRole.GuestOrAdministrator"Guest,Administrator"Guest or Administrator.

In the endpoint blocks below, the Any badge denotes the role set required by the action's Roles argument; where an action additionally restricts the policy to JWT or API key, that is stated in the notes. When the action specifies a policy but no explicit Roles, any authenticated principal that satisfies the policy is permitted. See Authentication & Security for token acquisition, API-key configuration, and the no-auth mode.

Source: src/slskd/Common/Authentication/AuthPolicy.cs, AuthRole.cs

URL-encoded path segments

Many path parameters carry a value that may contain reserved characters — most often a username or roomName. These parameters are decorated with the slskd-specific [UrlEncoded] attribute and bound by a custom model binder that reads the raw request target and applies a single UnescapeDataString pass. Callers must percent-encode (URL-encode) such segments exactly once. The web client does this with encodeURIComponent(...). This is distinct from base64 encoding (below).

Source: src/slskd/Common/Middleware/UrlEncodingModelBinder.cs

Base64-encoded path segments

The Files controller addresses local filesystem paths through path parameters named {base64SubdirectoryName} and {base64FileName}. The value is the path relative to the configured downloads/incomplete root, encoded with standard Base64 (the web client uses btoa(...)). On the server the value is Base64-decoded and both \ and / are normalized to the host directory separator before being safely combined with the root directory. No other controller uses Base64 path segments.

Source: src/slskd/Files/API/FilesController.cs, src/web/src/lib/files.js

Error responses & status codes

Validation and domain errors generally return a status code with a plain-text or JSON-string message body (for example, 400 with "Username and/or Password missing or invalid"). Unhandled exceptions surface as 500. Common codes used across the API:

StatusTypical meaning in slskd
200 OK / 201 Created / 204 No ContentSuccess (body present / created / no body).
205 Reset ContentServer connect: a connection attempt was already in progress and was restarted.
207 Multi-StatusDownload batch: some files enqueued, some failed.
304 Not ModifiedDeclared via ProducesResponseType on SearchesController.Cancel and RoomsController.JoinRoom but never actually returned. The real responses for an already-joined room or a cancelled/no-op search are 200 (404 if the resource is unknown).
400 Bad RequestMalformed input; message body describes the problem.
401 UnauthorizedMissing/invalid credentials, or failed login.
403 ForbiddenAuthenticated but not permitted; also returned when a feature is disabled (remote config/file management) or the daemon is running as a relay agent.
404 Not FoundResource absent, or user offline/blacklisted.
409 ConflictDuplicate id, or operation already in progress (e.g. share scan).
429 Too Many RequestsA serialized operation (search / download enqueue) is already running.
500 Internal Server ErrorUnexpected failure; message body contains the exception message.
Relay agent mode

When the daemon runs as a relay agent (relay.mode = agent), nearly every user-facing controller action short-circuits with 403 Forbidden via an internal Program.IsRelayAgent check. In agent mode the daemon's UI/API surface for searches, transfers, users, rooms, conversations, etc. is intentionally disabled. See Relay Protocol.

Pagination & filtering

Pagination is not uniform across the API. The endpoints that support it use offset + limit query parameters (Events, and the Telemetry reports leaderboard/exceptions/directories). The Events list additionally returns a total-record count in the X-Total-Count response header. Time-windowed report endpoints accept start/end (ISO-8601 DateTime, UTC) and frequently a direction and/or username filter. Most collection endpoints (transfers, searches, conversations) return the full set and rely on client-side filtering.

Reverse-proxy prefix (url_base)

When slskd is hosted behind a path prefix, the option web.urlBase is applied via ASP.NET UsePathBase. All routes documented here are then served beneath that prefix — e.g. with urlBase = /slskd, the searches list is GET /slskd/api/v0/searches. The URL-encoded model binder is prefix-aware. The web client computes its base URL as ${urlBase}/api/v0.

Source: src/slskd/Program.cs (UsePathBase), src/web/src/config.js

Application — /application

Application lifecycle, version, diagnostics. Maps to ApplicationController.

Get application state

GET/api/v0/applicationAny

Returns the current application State object (server connection, current user, version/update status, shares, relay, pending restart, etc.). See Data Model & Types and Real-Time API & Events (the same object is pushed over SignalR).

Shut down

DELETE/api/v0/applicationAdministrator

Cancels the master token, stops the host and exits the process (Environment.Exit(0)) after a 500 ms delay. Policy JwtOnly + Administrator role. Returns 204.

Restart

PUT/api/v0/applicationAdministrator

Starts a new instance of the executable with the same command line, then stops the host. Policy JwtOnly + Administrator. Returns 204.

Get version

GET/api/v0/application/versionAny

Returns the running semantic version (Program.SemanticVersion) as a JSON string.

Check for updates

GET/api/v0/application/version/latestAny
QueryTypeDefaultDescription
forceCheckboolfalseIf true, triggers a fresh remote version check before returning.

Returns the cached version/update state (latest version, whether an update is available).

Collect garbage

POST/api/v0/application/gcAny

Forces a .NET garbage collection. Returns 200.

Dump memory

GET/api/v0/application/dumpAny

Captures a process memory dump and returns it as a file download (application/octet-stream, filename slskd.dmp).

Loopback (diagnostic)

POST/api/v0/application/loopbackAny

Logs the posted JSON body at Information level and returns 200. A diagnostic no-op.

Source: src/slskd/Core/API/Controllers/ApplicationController.cs

Options — /options

Runtime configuration. The full Options object is documented in Data Model & Types; secrets are redacted (Redact()) on every read. Several actions require remoteConfiguration to be enabled and return 403 otherwise.

Get current options

GET/api/v0/optionsAny

Returns the current, redacted effective Options.

Apply options overlay

PATCH/api/v0/optionsAdministrator

Applies an in-memory configuration overlay (an OptionsOverlay JSON body) over the current options. Requires remoteConfiguration = true (else 403). Validates the overlay; on failure returns 400 with the validation message. On success returns the redacted overlay. A null body returns 204.

Get startup options

GET/api/v0/options/startupAny

Returns the redacted options captured at process start (OptionsAtStartup), before any overlay or hot-reload.

Get configuration debug view

GET/api/v0/options/debugAdministrator

Policy JwtOnly + Administrator. Returns the .NET configuration provider debug view (which key came from which source) as a string. Requires both debug = true (at startup) and remoteConfiguration = true; otherwise 403.

Get YAML file location

GET/api/v0/options/yaml/locationAdministrator

Policy JwtOnly + Administrator. Returns the absolute path of the active configuration file. Requires remoteConfiguration = true.

Get YAML file

GET/api/v0/options/yamlAdministrator

Policy JwtOnly + Administrator. Returns the raw contents of the configuration YAML file. Requires remoteConfiguration = true.

Update YAML file

PUT/api/v0/options/yamlAdministrator

Policy JwtOnly + Administrator. Body: the new YAML as a JSON string. Requires remoteConfiguration = true. Validates the YAML (deserialize + TryValidate); on failure returns 400 with the error. On success, backs up the existing file to <file>.bak, writes the new contents, and — if config-watch is disabled — sets pendingRestart. Returns 200.

Validate YAML

POST/api/v0/options/yaml/validateAny

Body: candidate YAML as a JSON string. Requires remoteConfiguration = true (else 403). Returns 200 with an empty body if valid, or 200 with the validation error string if invalid. (Note: validation failures are returned as 200 + message, not 400.)

Source: src/slskd/Core/API/Controllers/OptionsController.cs

Server — /server

Controls and reports the connection to the central Soulseek server. Forbidden in relay-agent mode.

Get server state

GET/api/v0/serverAny

Returns the ServerState (address, connection state, logged-in flag, etc.). See Data Model & Types.

Connect

PUT/api/v0/serverAny

Starts (or, if already running, restarts) the connection watchdog that establishes and maintains the server connection. Returns 200 on a fresh start, or 205 Reset Content if a connection attempt was already in progress and has been restarted. 403 in relay-agent mode.

Disconnect

DELETE/api/v0/serverAny

Body: an optional disconnect message as a JSON string. Stops the watchdog (aborting reconnect) and, if connected, disconnects with an IntentionalDisconnectException so the watchdog will not auto-reconnect. Returns 204. 403 in relay-agent mode.

Source: src/slskd/Core/API/Controllers/ServerController.cs

Transfers — /transfers

Downloads and uploads, plus the batch download API. State machine, queue semantics, and the Transfer shape are in Transfers & Queueing; this section documents the HTTP surface. The {username} segment is URL-encoded. Forbidden in relay-agent mode. Download enqueue (single and batch) is serialized via a 2-slot limiter; overflow returns 429.

List endpoints return data grouped by user then directory as UserResponse[] — each with username and a directories array of { directory, fileCount, files: Transfer[] }.

List all downloads

GET/api/v0/transfers/downloadsAny
QueryTypeDefaultDescription
includeRemovedboolfalseInclude soft-removed records.

Returns UserResponse[] grouped by user and directory.

List a user's downloads

GET/api/v0/transfers/downloads/{username}Any

Returns a single UserResponse; 404 if the user has no downloads.

Get a download

GET/api/v0/transfers/downloads/{username}/{id}Any

{id} is the download's uuid. Returns the Transfer; 400 if id is not a GUID; 404 if not found.

Get place in remote queue

GET/api/v0/transfers/downloads/{username}/{id}/positionAny

Requests the current place-in-queue of the download from the remote peer. Returns the updated Transfer; 404 if the download is unknown or the user is blacklisted; 400 on a bad GUID.

Cancel / remove a download

DELETE/api/v0/transfers/downloads/{username}/{id}Any
QueryTypeDefaultDescription
removeboolfalseAlso remove the tracked record after cancellation.

Returns 204; 404 if not found; 400 on a bad GUID.

Clear completed downloads

DELETE/api/v0/transfers/downloads/all/completedAny

Removes every non-removed download in a completed state (succeeded or failed). Returns 204; 500 on error.

Enqueue downloads (obsolete)

POST/api/v0/transfers/downloads/{username}Any

Obsolete — prefer the batch endpoint below. Body: an array of QueueDownloadRequest ({ filename, size }, filename required, size a int64). Validates that the list is non-empty, contains no nulls, and that no filename contains path-traversal segments. Returns 201 with { "enqueued": [...], "failed": [...] }. Empty/invalid list → 400; overflow → 429; user offline → handled as error; unexpected → 500.

POST /api/v0/transfers/downloads/RemoteUser
[ { "filename": "@@abc\\Music\\track.flac", "size": 41557123 } ]

Enqueue a download batch

POST/api/v0/transfers/downloads/batchesAny

The preferred enqueue API. Request body QueueDownloadBatchRequest:

FieldTypeReq.Description
idstring (uuid)—Batch id; generated if omitted. Must be a valid GUID when present.
searchIdstring (uuid)—Associated search id, if any.
usernamestringyesSource user (1–500 chars, non-blank).
filesEnqueueDownloadBatchItem[]yes≥1 item; filename (required, non-blank, non-traversing) and size (int64, required, ≥0). Filenames must be unique within the batch.
optionsEnqueueDownloadBatchOptions—destination (relative, non-traversing subdir under the downloads root) and an optional externalId.

Events — /events

The persisted event log and a test harness for raising synthetic events. Live event delivery is over SignalR — see Real-Time API & Events for the event model and types.

List events (paginated)

GET/api/v0/eventsAny
QueryTypeDefaultDescription
offsetint320Record offset (≥0).
limitint32100Page size (>0).

Returns EventRecord[] and sets the X-Total-Count response header to the total number of records. 400 if offset < 0 or limit ≤ 0.

Raise a sample event

POST/api/v0/events/{type}Any

{type} is an EventType name (case-insensitive). The six types that successfully generate a sample and return 201 are DownloadFileComplete, DownloadDirectoryComplete, UploadFileComplete, PrivateMessageReceived, RoomMessageReceived, Noop. Body: an optional disambiguator string used to vary the generated sample. Raises a randomly-populated event of that type onto the event bus and returns 201 with the generated Event. The sentinel types None and Any cannot be raised → 400; an unknown type → 400 with the list of valid names.

Caveat

The EventType enum also defines SoulseekClientConnected and SoulseekClientDisconnected, and the 400 "valid names" message (built from Enum.GetNames minus Any/None) advertises them as accepted. However, they parse past the guard and then hit the sample-generation switch's default case, which throws → 500 rather than producing a sample. Only the six types listed above actually yield a 201.

Note

The "loopback" diagnostic endpoint lives on the Application controller — POST /api/v0/application/loopback — not on Events.

Relay — /relay

HTTP surface of the relay subsystem (RelayController). Protocol semantics, the agent/controller roles, and credential computation are in Relay Protocol. These endpoints are gated by the relay mode and enabled flag and return 403 when not applicable.

Note

Agent registration is not a REST call: an agent registers with the controller over the SignalR relay hub (RelayHub), not via /relay. The controller/* endpoints below are the data-plane that an agent calls back into. See Relay Protocol.

Agent: connect to controller

PUT/api/v0/relay/agentAdministrator

Policy JwtOnly + Administrator. Starts the agent's relay client (connecting to the configured controller). Allowed only when relay is enabled and the mode is Agent or Debug; otherwise 403. Returns 200.

Agent: disconnect from controller

DELETE/api/v0/relay/agentAdministrator

Policy JwtOnly + Administrator. Stops the agent's relay client. Same mode gate as above. Returns 204.

Controller: serve a file to the agent

GET/api/v0/relay/controller/downloads/{token}Any

Policy ApiKeyOnly. Called by an agent to download a completed file from the controller. {token} is the request GUID. Required headers: X-Relay-Agent (agent name), X-Relay-Credential (encrypted credential), X-Relay-Filename-Base64 (Base64 of the requested filename). The controller validates the credential and streams the file as application/octet-stream. Invalid token format → 400; unknown agent / missing credential / failed validation → 401; missing or path-traversing filename → 400. The token remains valid after use to support retries.

Controller: receive a file from the agent

POST/api/v0/relay/controller/files/{token}ReadWrite

Policy ApiKeyOnly + ReadWrite/Administrator. Multipart form upload of a single file stream from the agent, used to feed an outgoing upload in flight (request size limit 10 TiB). Requires X-Relay-Agent and X-Relay-Credential headers; the filename is taken from the multipart content-disposition. Non-multipart → 415; unknown agent / failed credential validation → 401; bad/missing/path-traversing filename → 400. Returns 200 when the forwarded upload completes.

Controller: receive shares from the agent

POST/api/v0/relay/controller/shares/{token}ReadWrite

Policy ApiKeyOnly + ReadWrite/Administrator. Multipart form upload of the agent's share catalogue: a shares form field (JSON array of Share) plus the share database file (request size limit 1 TiB). Requires X-Relay-Agent and X-Relay-Credential headers. Non-multipart → 415; unknown agent / failed validation → 401; malformed form or share-validation failure → 400. Returns 200 on success.
